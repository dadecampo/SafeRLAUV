Detailed Description

Main Smoke & Fire solver component

Each Smoke & Fire component corresponds to one instance of simulation. Different instances of simulation can't interact with each other.

Some parameters can't be after simulation has started and we created GPU buffers. Normally, simulation starts in playmode in OnEnable and stops in OnDisable. To change those parameters in runtime you want to have this component disabled, and after setting them, enable this component.

OnEnable will allocate GPU buffers, which may cause stuttering. Consider enabling simulation volume on level load, but with simulation/render paused, to not pay the cost of fluid initialization during gameplay.

Disabling simulation volume will free GPU buffers. This means that Smoke&Fire state will be lost.

Various parameters of the simulation volume are spread throught multiple components. This is done so you can use Unity's Preset system to only change part of parameters.

◆ CurrentSimulationMode

SimulationMode CurrentSimulationMode = SimulationMode.Fire

Setting that determines the type of simulation being performed, with options including Smoke, Colored Smoke, and Fire.

Smoke mode simulates a single colored smoke/fog/etc.

Colored Smoke mode allows emitting smoke of a given color.

Fire mode simulates smoke, fuel, and temperature components, allowing control of burning fuel to produce fire.

Depending on this parameter, simulation will use different parameters defined in various other classes.

Changing this parameter during simulation has no effect. See ActiveSimulationMode.

◆ MaxFramesInFlight

UInt32 MaxFramesInFlight = 3

Maximum allowed number of frames queued to render.

Only used when QualitySettings.maxQueuedFrames is not available or invalid.

Defines number of frames we'll wait between submitting simulation workload and reading back simulation information back to the CPU. Higher values correspond to more delay for simulation info readback, while lower values can potentially decreasing framerate.

◆ SimulationIterations

int SimulationIterations = 3

Number of simulation iterations per simulation frame.

The simulation does 1/3 of the smoke simulation per iteration, and extrapolates the smoke movement in time for higher performance while keeping smooth movement. To do a full simulation per frame you can set it to 3 iterations, which may be beneficial in cases where the simulation interacts with quickly moving objects.
