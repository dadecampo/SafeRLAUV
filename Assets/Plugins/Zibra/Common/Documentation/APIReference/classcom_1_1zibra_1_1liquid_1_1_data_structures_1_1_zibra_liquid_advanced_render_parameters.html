Detailed Description

Component that contains raymarching and mesh generation liquid parameters.

It doesn't execute anything by itself, it is used by ZibraLiquid instead.

It's separated so you can save and apply presets for this component separately.

Disables liquid raymarching

◆ MaxLiquidMeshSize

float MaxLiquidMeshSize = 1.0f

Determines how much memory to allocate for liquid mesh.

This parameter noticeable affects VRAM cost. When using Unity Render mode, it will also heavily affect performance.

Since liquid mesh can take vastly different amount of memory, best value will depend on specific usecase. Generally, the more surface area liquid has, the higher value you need. Value of 3.0 represents largest possible liquid mesh, while values closer to 0 represent much smaller meshes. If you set this value too low, you will get artifacts that some triangles of the liquid will dissapear, but it is safe to get such artifacts since it won't result in any additional issues/bugs/crashes.

When using Mesh Render mode, mesh is rendered in Native Plugin using current actual number of vertices of generated mesh. When using Unity Render mode, we can not render variable number of vertices, and instead forced to use maximum number of vertices, most of which are potentially discarded. So in Unity Render mode, this parameter directly corresponds to number of vertices rendered by liquid.

◆ VertexOptimizationIterations

int VertexOptimizationIterations = 5

Determines number of vertex optimization iterations.

Each iteration improves mesh quality by moving liquid mesh closer to actual liquid surface. Higher values correspond to less blocky mesh but lower performance.

To get voxel like mesh, set this, and other mesh/vertex optimization iterations to 0.

◆ DualContourIsoSurfaceLevel

float DualContourIsoSurfaceLevel = 0.025f

Defines the density threshold used for the mesh generation.

When generating liquid mesh, first, we generate liquid density grid. This value specifies which threshold which defines which parts of the grid are inside or outside the liquid, if the density is larger than this value then the grid node is assumed to be inside. Then mesh is generated on the edge of liquid in the grid.

◆ IsoSurfaceLevel

float IsoSurfaceLevel = 0.36f

Defines the density threshold used for the vertex optimization.

For best quality, it's recommended to have this value somewhat larger compared to DualContourIsoSurfaceLevel.

During vertex optimization, vertices are moved to nearest point with specified density.

◆ RayMarchIsoSurface

float RayMarchIsoSurface = 0.65f

Defines the density threshold used for the raymarching.

For best quality, it's recommended to have this value less or equal compared to IsoSurfaceLevel.

During raymarching, collision with liquid surface considered to be transition between values of density separated by this threshold.

Has no effect when DisableRaymarch is enabled.

int RayMarchMaxSteps = 128

Defines the max number of raymrach steps.

Max number of steps we should do, after which we should stop raymarching. Higher values correspond to higher performance cost, but setting this too low may result in artifacts that make liquid opaque in some pixels.

float RayMarchStepSize = 0.2f

Defines the distance for each raymarch step.

Distance for each raymarch step determines accuracy of refraction. Lower values correspond to higher quality, but setting this too low may result in artifacts for some pixels, making liquid opaque.
