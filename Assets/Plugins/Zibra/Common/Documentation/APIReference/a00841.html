Detailed Description

Component that contains liquid physics behaviour parameters.

It doesn't execute anything by itself, it is used by ZibraLiquid instead.

It's separated so you can save and apply presets for this component separately.

◆ FluidStiffness

float FluidStiffness = DEFAULT_STIFFNESS

Stiffness of the liquid.

Due to performance reasons, our liquid is somewhat compressible. This parameters defines how resistant to compression liquid is. Lower values correspond to more compressible liquid. Higher values may result in unstable liquid.

◆ ParticleDensity

float ParticleDensity = 2.0f

Resting density of the liquid particles measured in particles/grid node.

This option directly affects volume of the liquid. Higher values correspond to lower volume of the liquid, but higher quality simulation.

Since liquid is somewhat compressible, this parameter won't match liquid state exactly.

◆ Viscosity

float Viscosity = DEFAULT_VISCOSITY

Viscosity of the liquid.

Due to simulation algorithm used, maximum visosity is limited. You can't push it too far to make liquid honey like.

◆ SurfaceTension

float SurfaceTension = 0.0f

Surface tension of the liquid.

Normally, liquids have very low, positive surface tension.

Can be set to negative values. This will force liquid to spread and separate into many droplets.

◆ MaximumVelocity

float MaximumVelocity = DEFAULT_MAX_VELOCITY

Maximum allowed velocity for particles.

In cases of high or unstable simulation timesteps, it's recommended to not set it to higher values.

Can be set to 0 to freeze the liquid, but colliders will still be able to push the liquid.

◆ MinimumVelocity

float MinimumVelocity = 0.0f

Minimum allowed velocity for particles.

Normally, it should be 0. Only set it to other values if you want to force liquid to perpetually move.

Should not exceed MaximumVelocity

◆ ForceInteractionStrength

Has logarithmic scale. Value of 0 corresponds to defualt intraction strenght. Other values will scale strentgh by exp(ForceInteractionStrength).

◆ HeightmapResolution

Defines the resolution of the heightmaps used in heightmap SDFs and terrain SDFs

Each heightmap in heightmap based SDFs are resampled into this resolution. For best quality you should use a resolution which is at least the resolution of the smallest heightmap used
